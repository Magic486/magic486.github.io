obsidian插入图片教程

发表于2026-02-05
|浏览量:

这里采用的是Obsidian + PicGo + 图床 的方法,好处是想要插入图片是直接粘贴进obsdian即可,而更新博客时也是直接复制文章即可.
这里图床的选择有Gitee和Github
Gitee国内可以直连，速度快，但是单仓库容量小，长期使用会爆满，并且有反盗链，博客里无法查看图片
GitHub上传图片需要科学上网，但是单仓库容量大，配合jsDelivr (CDN 加速)在国内无需梯子也可以快速访问

详细步骤

第一步：获取 Gitee 的私人令牌 (Token)

  1. 登录 Gitee 官网。

  2. 点击右上角头像 -> 设置。

  3. 在左侧菜单栏找到 安全设置 -> 私人令牌。

  4. 点击 生成新令牌。

  • 私人令牌描述：填入“Obsidian_PicGo”等便于识别的名字。

  • 权限选择：只需勾选 **projects**（项目权限）即可。

  1. 点击提交，输入密码后会显示一段字符串。请务必立即保存该 Token，关闭网页后将无法再次查看。

第二步：配置 PicGo 软件

  1. 安装 PicGo：确保你已安装 PicGo 客户端。

  2. 安装 Gitee 插件：

  • PicGo 默认不内置 Gitee 图床，点击左侧的 插件设置。
  • 在搜索栏输入 gitee，安装常用插件（如 gitee-uploader）。
  • 注意：安装插件可能需要你电脑预装了 Node.js。
  1. 设置 Gitee 图床：
  • 点击左侧 图床设置 -> Gitee（安装插件后才会出现）。
  • repo：填写 用户名/仓库名。
  • branch：通常填写 master。
  • token：粘贴刚才在 Gitee 生成的私人令牌。
  • path：填写图片存储的文件夹路径，例如 img/（不填则存放在根目录）。
  • customPath 和 customUrl：一般保持默认即可。
  1. 设为默认图床：点击确定并设为默认。

第三步：配置 Obsidian 插件

为了实现“粘贴即上传”，你需要在 Obsidian 里配置 Image Auto Upload Plugin：

  1. 安装插件：在 Obsidian 第三方插件 市场搜索 Image Auto Upload Plugin 并安装启用。

  2. 连接 PicGo：

    • 进入该插件设置界面。

    • Uploader：选择 PicGo(app)。

    • PicGo Server：默认为 http://127.0.0.1:36677。

  3. 确认 PicGo 设置：

    • 打开 PicGo 软件设置 -> 设置 Server。

    • 确保 开启 Server 按钮是打开的，且 监听端口 是 36677。

第四步：测试

  1. 在 Obsidian 中按下快捷键截图，或者复制一张网络图片。

  2. 在笔记页面按下 Ctrl + V 粘贴。

  3. 如果配置成功，你会看到图片链接从本地路径（或 uploading... 提示）瞬间变为 https://gitee.com/xxx/xxx/raw/master/xxx.png。

可能会遇到的问题

插件设置里面找不到gitee

通常是由于 PicGo 无法正确调用 npm 路径，或者是国内访问 npm 官方源的网络延迟导致的

==解决方法==

通过命令行手动安装插件

  • 按下 Win + R，输入 cmd 回车。
  • 输入以下命令并回车（这会自动下载并安装 gitee 插件到 PicGo 的插件目录）
  • 重启 PicGo。此时点击左侧的“图床设置”，你应该能看到 Gitee 选项了。

如果使用Github

第一步：在 GitHub 创建仓库 (存图片的地方)

  1. 登录：打开 GitHub 并登录你的账号。

  2. 新建仓库：

    • 点击右上角的 + 号 -> 选择 New repository。

    • Repository name：起个名字，比如 obsidian-images。

    • Public/Private：**必须选择 Public (公开)**。

      • 注意：如果选 Private，jsDelivr 加速会失效，别人看你博客图也会挂。

    • 勾选 Add a README file (这是一个好习惯，能保证仓库初始化成功，有 main 分支)。

    • 点击 Create repository。

第二步：获取访问令牌 (Token) (PicGo 的“钥匙”)

PicGo 需要这个钥匙才能帮你上传图片。

  1. 点击 GitHub 右上角头像 -> Settings。

  2. 在左侧侧边栏最底部找到 Developer settings。

  3. 点击 Personal access tokens -> 选择 Tokens (classic) (不要选 Beta)。

  4. 点击 Generate new token -> **Generate new token (classic)**。

  5. Note：随便填，比如 PicGo。

  6. Expiration (过期时间)：建议选 No expiration (永不过期)，免得以后突然不能传了要重设。

  7. Select scopes (权限)：这是最重要的一步！

    • 找到 repo 这一项，把前面的框勾上 (这就包含了上传文件所需的所有权限)。

  8. 拉到最底下点击 Generate token。

  9. 立刻复制那一串 ghp_ 开头的字符。

    • 警告：它只显示一次，如果你关掉页面就再也看不到了（只能重新生成）。

第三步：配置 PicGo

  1. 打开 PicGo 软件。

  2. 点击左侧 图床设置 -> 选择 GitHub。

  3. 按照以下格式填写：

    • 设定仓库名：填 用户名/仓库名。

      • 例如你的用户名是 magic486，仓库叫 obsidian-images，就填：magic486/obsidian-images

    • 设定分支名：填 main (现在的 GitHub 默认分支通常是 main，不再是 master 了)。

    • 设定 Token：粘贴刚才复制的那串 ghp_xxxx。

    • 指定存储路径：(可选) 填 img/，这样图片会整齐地放在 img 文件夹里。

    • 设定自定义域名：这里是加速的关键！

      • 填入：https://cdn.jsdelivr.net/gh/你的用户名/你的仓库名

      • 例如：https://cdn.jsdelivr.net/gh/magic486/obsidian-images

      • 注意：末尾不要加 /

  4. 点击 确定，然后点击 设为默认图床。
